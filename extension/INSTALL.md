# V1 Helper — Chrome Extension Install

## Install (Developer Mode)

1. Open Chrome and navigate to `chrome://extensions`
2. Enable **Developer mode** (toggle in top-right corner)
3. Click **Load unpacked**
4. Select the `extension/` directory from this repository
5. The V1 Helper icon appears in the Chrome toolbar

## Usage

### Import CVE Analysis
1. Click the V1 Helper icon in the toolbar
2. Click **Import analysis.json**
3. Select your analysis file (generated by `python scripts/report_generator.py`)
4. CVE counts appear in the popup (Relevant / Low / Not Relevant)

### View Overlays on V1
1. Navigate to Vision One > Cloud Security > Container Security > Vulnerabilities
2. CVE rows automatically get color-coded badges:
   - **Red (RELEVANT)** — affects runtime packages, needs action
   - **Yellow (LOW)** — transitive dependency, monitor only
   - **Green (NOT RELEVANT)** — wrong platform, unreachable code, or kernel-only
3. Click any badge for detailed analysis (summary, relevance reasoning, remediation)

### Toggle Overlays
- Click the V1 Helper icon > **Overlays On/Off** button

### V1 API Settings (optional)
- Click **Settings** in the popup
- Enter a V1 API key and select region
- Click **Test Connection** to verify
- Add customer context for relevance reasoning

## Generating analysis.json

```bash
# Full run (pulls V1 API data + runs Claude analysis)
python scripts/report_generator.py --customer <name>

# The analysis file is saved to reports/<name>-analysis.json (or reports/analysis.json)
# Import this file into the extension
```

## Troubleshooting

- **Extension not loading:** Check `chrome://extensions` for errors. Click the reload button.
- **No overlays on V1 page:** Verify you're on a Container Security > Vulnerabilities page. Check that overlays are enabled in the popup.
- **Import fails:** Ensure the file is valid JSON with CVE entries (object with CVE IDs as keys, or array with `cve` field per entry).
